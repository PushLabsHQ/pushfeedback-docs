Styles configuration

PushFeedback allows users to seamlessly integrate feedback mechanisms into their web pages while providing the flexibility to style and customize its appearance. This reference document lists all the available CSS custom properties that can be overridden to match your website's design.

Usage​

To customize these properties, define them in your website's CSS using the :root selector for global changes or a specific selector to target particular areas:

css Copy code :root { --feedback-primary-color: #00D68F; ... }

Properties​

The table below lists all the available properties you can override:

PropertyDefaultDescription
--feedback-primary-color#0070F4Primary color used throughout the feedback module.
--feedback-secondary-color#fbfbfbSecondary color for hover states and highlights.
--feedback-light-color#cccColor used for borders and separators.
--feedback-dark-color#191919Darker color used for primary text.
--feedback-text-color#5f6368Default color for general text elements.
--feedback-white-color#fffPure white color, often used for backgrounds.
--feedback-font-family"Tahoma", "sans-serif"Font family for text elements in the feedback module.
--feedback-heading-font-size16pxFont size for headings within the module.
--feedback-text-font-size14pxFont size for general text elements.
--feedback-modal-content-bg-colorvar(--feedback-white-color)Background color of the modal content.
.........
--feedback-modal-screenshot-header-bg-colorvar(--feedback-primary-color)Background color of the screenshot header within the modal.
