Sphinx

This guide will walk you through adding the PushFeedback button to your Sphinx-generated documentation.

Prerequisites​

  • A Sphinx project set up and running.
  • Basic knowledge of reStructuredText and Sphinx.
  • Your PushFeedback project ID.

1. Update the Sphinx Configuration​

All Sphinx configurations reside in the conf.py file. We'll be adding our scripts and styles here.

a. Add PushFeedback Styles​

Append the following code to conf.py to add the required stylesheet:

    'https://cdn.jsdelivr.net/npm/pushfeedback/dist/pushfeedback/pushfeedback.css',
]

If you already have other CSS files in the list, simply add the PushFeedback CSS link to the existing list.

b. Add PushFeedback Script​

Similarly, append the following code to conf.py to add the necessary script:

    'https://cdn.jsdelivr.net/npm/pushfeedback/dist/pushfeedback/pushfeedback.esm.js',
]

Again, if you already have an html_js_files list, just add the new script to that list.

2. Add the Feedback Button Component​

The easiest way to add the PushFeedback button to every page in a Sphinx site is to customize the layout template.

a. Extend the Basic Layout​

If you haven't already, you'll need to create a _templates directory within your Sphinx project's source directory. Then, inside _templates, create a new file named layout.html.

In layout.html, add:


{% block footer %}
{{ super() }}
<feedback-button project="<YOUR_PROJECT_ID>">Send feedback</feedback-button>
{% endblock %}

Replace <YOUR_PROJECT_ID> with your specific PushFeedback project ID.
